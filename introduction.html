Introducing Capabilities

Object oriented programming starts with a single object. Functional programming starts with a single function. The power in these systems comes from being able to build and compose powerful structures out of these primitives. Capability-based systems start out with a single capability, which is defined in Wikipedia as " describes a transferable right to perform one (or more) operations on a given object." This definition is vague in itself, but fortunately there’s another definition that will be explained in the context of capabilities implemented in the Scala programming language.

A capability is sufficient justification to affect a resource.
